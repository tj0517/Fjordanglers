# CLAUDE.md — FjordAnglers

Fishing guide marketplace connecting Nordic fishing guides with European anglers.

## Tech Stack

- **Framework:** Next.js 14+ (App Router)
- **Language:** TypeScript (strict mode)
- **Database:** Supabase (PostgreSQL + Auth + Realtime + Storage)
- **Payments:** Stripe Connect (Accounts v2 + Express Dashboard, Destination Charges) + Direct Payment tier (booking fee only)
- **Hosting:** Vercel
- **Styling:** Tailwind CSS
- **Email:** Resend (transactional emails)
- **Scheduling:** Vercel Cron Jobs + Supabase pg_cron

## Stripe Architecture

### Model: Destination Charges + Accounts v2 (Express Dashboard)

FjordAnglers is the **merchant of record**. All payments go through the platform account, with automatic transfers to guide's connected accounts.

**IMPORTANT:** We use **Accounts v2 API** (NOT legacy `type: "express"`). The old account types (Standard/Express/Custom) are legacy. Use controller properties instead.

### Key Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Account API | **Accounts v2** with controller properties | Modern API, replaces legacy Express/Standard/Custom types |
| Dashboard | `controller.stripe_dashboard.type: "express"` | Guide sees Express Dashboard (balance, payouts, earnings) |
| Fee responsibility | `controller.fees.payer: "application"` | Platform pays Stripe processing fees, guide never sees them |
| Loss liability | `controller.losses.payments: "application"` | Required for Express Dashboard + destination charges |
| Onboarding | `controller.requirement_collection: "stripe_hosted"` | Stripe handles KYC, identity verification, bank account |
| Charge type | Destination charges | Platform is MoR, controls refunds & disputes |
| Checkout | Stripe Checkout (hosted) | PCI compliant, supports all payment methods, mobile-optimized |
| Payout schedule | Manual | Platform controls timing (24-48h after trip completion) |
| Fee model | `application_fee_amount` | Commission + service fee collected as single application fee |

### Commission Structure

```typescript
// lib/stripe/fees.ts

const FOUNDING_GUIDE_COMMISSION = 0.08;  // 8% — first 50 guides, 24 months
const STANDARD_GUIDE_COMMISSION = 0.10;  // 10% — standard rate
const CUSTOMER_SERVICE_FEE = 0.05;       // 5% — charged to angler on top of trip price
const CUSTOMER_SERVICE_FEE_CAP_CENTS = 5000; // €50.00 — max service fee regardless of trip price

function calculateServiceFee(tripTotalCents: number): number {
  const calculated = Math.round(tripTotalCents * CUSTOMER_SERVICE_FEE);
  return Math.min(calculated, CUSTOMER_SERVICE_FEE_CAP_CENTS);
}

function calculateApplicationFee(
  tripPriceCents: number,
  guestCount: number,
  isFoundingGuide: boolean
): number {
  const total = tripPriceCents * guestCount;
  const commissionRate = isFoundingGuide ? FOUNDING_GUIDE_COMMISSION : STANDARD_GUIDE_COMMISSION;
  const commission = Math.round(total * commissionRate);
  const serviceFee = calculateServiceFee(total);
  return commission + serviceFee;
}

// IMPORTANT: application_fee_amount includes BOTH commission and service fee
// The service fee is added to line_items as a separate line item for transparency
// But Stripe's application_fee_amount handles the actual money split
//
// Service fee cap examples:
//   Trip €500  → service fee = €25.00 (5%)
//   Trip €800  → service fee = €40.00 (5%)
//   Trip €1000 → service fee = €50.00 (5% = cap)
//   Trip €2000 → service fee = €50.00 (capped, not €100)
```

### Stripe Fee Responsibility

- Stripe processing fee (~1.5% EU + €0.25): **paid by platform** from application_fee revenue
- Platform NEVER charges Stripe fees to guides
- Guide always receives: `trip_price - commission` (nothing else deducted)

## Payment Tiers (guide-selectable)

Guides choose their payment tier during onboarding. They can upgrade at any time.

### Tier Overview

| Tier | Who processes trip payment? | Guide provides | Platform touches guide's money? | Available countries |
|---|---|---|---|---|
| **Stripe Connect** | Stripe (automatic split) | Stripe onboarding | No — Stripe handles split | NO, SE, DK, FI |
| **Direct Payment** | Angler pays guide directly | Nothing (optionally IBAN for SEPA QR) | **No — never** | All (incl. IS) |

### Tier 1: Stripe Connect (full)

Best experience. Angler pays 100% online. Guide gets Express Dashboard, automatic payouts, full transparency. Platform is MoR via destination charges. See [Stripe Architecture](#stripe-architecture) section for details.

### Tier 2: Direct Payment (booking fee only) ⭐ DEFAULT for new guides

Platform only charges angler the **booking fee** (commission + service fee). Guide receives the rest **directly from angler** — cash, bank transfer, terminal, Vipps, whatever guide accepts.

**Platform never touches guide's money.** Zero PSD2 risk, zero money transmission.

```typescript
// lib/stripe/fees.ts — Direct Payment tier

type PaymentTier = "stripe_connect" | "direct_payment";

// Direct Payment: angler pays booking fee to platform, rest to guide directly
function calculateDirectPaymentAmounts(
  tripPriceCents: number,
  guestCount: number,
  isFoundingGuide: boolean
): {
  bookingFeeCents: number;     // Angler pays this to platform via Stripe
  guideAmountCents: number;    // Angler pays this directly to guide
  anglerTotalCents: number;    // Total angler pays (booking fee + guide amount)
  commissionCents: number;     // Platform's commission portion
  serviceFeeCents: number;     // Service fee portion
} {
  const tripTotal = tripPriceCents * guestCount;
  const commissionRate = isFoundingGuide ? FOUNDING_GUIDE_COMMISSION : STANDARD_GUIDE_COMMISSION;
  const commission = Math.round(tripTotal * commissionRate);
  const serviceFee = calculateServiceFee(tripTotal);

  const bookingFee = commission + serviceFee;
  const guideAmount = tripTotal - commission;  // Guide gets trip price minus commission

  return {
    bookingFeeCents: bookingFee,
    guideAmountCents: guideAmount,
    anglerTotalCents: guideAmount + bookingFee,  // = tripTotal + serviceFee
    commissionCents: commission,
    serviceFeeCents: serviceFee,
  };
}

// Example: Trip €500, founding guide (8%), 1 guest
//   commission  = €40.00 (8% of €500)
//   serviceFee  = €25.00 (5% of €500, under €50 cap)
//   bookingFee  = €65.00 (angler pays platform via Stripe)
//   guideAmount = €460.00 (angler pays guide directly)
//   anglerTotal = €525.00 (same as Stripe Connect tier!)
//
// Example: Trip €2000, standard guide (10%), 1 guest
//   commission  = €200.00 (10% of €2000)
//   serviceFee  = €50.00 (5% = €100, but CAPPED at €50)
//   bookingFee  = €250.00 (angler pays platform via Stripe)
//   guideAmount = €1800.00 (angler pays guide directly)
//   anglerTotal = €1850.00
```

#### Direct Payment — Checkout Flow

1. Angler selects trip and clicks "Book"
2. Checkout page shows: "Booking fee: €65 (paid now) + Trip payment: €460 (paid directly to guide)"
3. Angler completes Stripe Checkout for **booking fee only** (€65)
4. Booking confirmed → emails sent to both parties
5. Angler receives guide's payment info (SEPA QR code + IBAN + guide's preferred methods)
6. Angler pays guide directly before or on day of trip
7. Guide confirms receipt in dashboard (or trip marked as completed)

#### Direct Payment — Guide's Payment Info to Angler

Platform automates sharing guide's payment details with angler:

```typescript
// lib/utils/sepa-qr.ts — Generate EPC QR Code for bank transfer

function generateEpcQrPayload(
  guideName: string,
  guideIban: string,
  amountEur: number,
  bookingRef: string
): string {
  // EPC 069-12 standard — supported by all EU/EEA banking apps
  return [
    "BCD",                              // Service tag
    "002",                              // Version
    "1",                                // Encoding (UTF-8)
    "SCT",                              // SEPA Credit Transfer
    "",                                 // BIC (optional)
    guideName,                          // Beneficiary name (max 70 chars)
    guideIban,                          // IBAN
    `EUR${amountEur.toFixed(2)}`,       // Amount
    "",                                 // Purpose code
    bookingRef,                         // Reference (max 35 chars)
    `FjordAnglers Trip ${bookingRef}`,  // Remittance info
  ].join("\n");
}

// Guide can also specify preferred payment methods in their profile:
// - SEPA bank transfer (IBAN — auto-generates QR code)
// - Cash on arrival
// - Vipps (phone number)
// - MobilePay (phone number)
// - PayPal (email)
// - Card terminal on site
```

#### Direct Payment — Cancellation & Refunds

Platform only controls the booking fee. Guide's portion is between angler and guide.

| Scenario | Booking fee (platform) | Guide's portion |
|---|---|---|
| Angler cancels in free window | Full refund of booking fee | Guide returns any prepayment (not platform's responsibility) |
| Angler cancels in partial window | 50% refund of booking fee | Between angler and guide |
| Angler no-show | No refund | Guide keeps whatever was prepaid |
| Guide cancels (any reason) | Full refund of booking fee | Guide returns any prepayment |
| Force majeure | Full refund of booking fee | Guide returns any prepayment |

**IMPORTANT:** Platform's Terms of Service clearly state: "FjordAnglers facilitates the booking and charges a non-refundable booking fee. The trip payment is a direct transaction between angler and guide. FjordAnglers is not responsible for the trip payment."

#### Direct Payment — Invoicing

- **Platform → Angler:** Receipt/invoice for booking fee (€65) — "Booking & service fee, Trip #XYZ"
- **Guide → Angler:** Guide's own invoice for trip amount (€460) — guide's responsibility
- **Platform → Guide:** No invoice. Platform does not charge guide. Commission is embedded in booking fee paid by angler.

## Cancellation & Refund Engine

### Three Policies (guide-selectable)

```typescript
type CancellationPolicy = "flexible" | "moderate" | "strict";

interface RefundCalculation {
  refundPercent: number;     // 0, 50, or 100
  anglerRefundCents: number;
  guidePayoutCents: number;
  platformFeeCents: number;
}

// lib/utils/cancellation.ts
function calculateRefund(
  policy: CancellationPolicy,
  tripDateUTC: Date,
  cancellationDateUTC: Date,
  totalPriceCents: number,
  serviceFeeCents: number,
  commissionRate: number
): RefundCalculation {
  const hoursUntilTrip = differenceInHours(tripDateUTC, cancellationDateUTC);

  let refundPercent: number;

  switch (policy) {
    case "flexible":
      if (hoursUntilTrip > 48) refundPercent = 100;
      else if (hoursUntilTrip > 0) refundPercent = 50;
      else refundPercent = 0; // no-show
      break;
    case "moderate":
      if (hoursUntilTrip > 168) refundPercent = 100;      // > 7 days
      else if (hoursUntilTrip > 48) refundPercent = 50;
      else refundPercent = 0;
      break;
    case "strict":
      if (hoursUntilTrip > 336) refundPercent = 100;      // > 14 days
      else if (hoursUntilTrip > 168) refundPercent = 50;   // > 7 days
      else refundPercent = 0;
      break;
  }

  // ... calculate actual amounts based on refundPercent
}
```

### Refund Stripe API Mapping

| Scenario | `reverse_transfer` | `refund_application_fee` |
|---|---|---|
| 100% refund (angler cancels in free window) | `true` | `true` |
| 50% refund (angler cancels in partial window) | `true` | `false` — recalculate fee on retained amount |
| 0% refund (no-show) | N/A | N/A — no refund |
| Guide cancels (weather/safety) | `true` | `true` |
| Guide cancels (personal reason) | `true` | `true` + apply guide penalty |
| Force majeure | `true` | `true` |

## Webhook Handler Pattern

```typescript
// app/api/webhooks/stripe/route.ts

import { headers } from "next/headers";
import Stripe from "stripe";
import { stripe } from "@/lib/stripe/client";
import { createAdminClient } from "@/lib/supabase/admin";

export async function POST(req: Request) {
  const body = await req.text();
  const signature = headers().get("stripe-signature")!;

  let event: Stripe.Event;

  try {
    event = stripe.webhooks.constructEvent(
      body,
      signature,
      process.env.STRIPE_WEBHOOK_SECRET!
    );
  } catch (err) {
    console.error("Webhook signature verification failed:", err);
    return new Response("Invalid signature", { status: 400 });
  }

  const supabase = createAdminClient();

  switch (event.type) {
    case "checkout.session.completed":
      await handleCheckoutCompleted(event.data.object, supabase);
      break;
    case "payment_intent.succeeded":
      await handlePaymentSucceeded(event.data.object, supabase);
      break;
    case "payment_intent.payment_failed":
      await handlePaymentFailed(event.data.object, supabase);
      break;
    case "charge.refunded":
      await handleRefund(event.data.object, supabase);
      break;
    case "payout.paid":
      await handlePayoutPaid(event.data.object, supabase);
      break;
    case "payout.failed":
      await handlePayoutFailed(event.data.object, supabase);
      break;
    case "charge.dispute.created":
      await handleDisputeCreated(event.data.object, supabase);
      break;
  }

  return new Response("OK", { status: 200 });
}
```

```typescript
// /api/cron/charge-balances logic:
// 1. Query bookings WHERE status = 'deposit_paid'
//    AND transaction_type = 'deposit_auto_charge'
//    AND balance_charge_date <= today
//    AND balance_charge_attempts < 3
// 2. For each: create off-session PaymentIntent with saved payment method
// 3. On success: update status to 'fully_paid'
// 4. On failure: increment balance_charge_attempts, notify angler
// 5. If attempts >= 3: cancel booking, trigger refund per cancellation policy

// /api/cron/trigger-payouts logic:
// 1. Query bookings WHERE status = 'fully_paid' OR status = 'trip_completed'
//    AND trip_date < now() - interval '24 hours'
//    AND payout_status = 'pending'
// 2. For each: create Payout on guide's connected account
// 3. Update payout_status to 'processing'
// 4. Webhook payout.paid → update to 'paid'
```

## Coding Rules

1. **All money in cents** — never use floats for currency. `350.00 EUR` = `35000` cents.
2. **Always round with `Math.round()`** after percentage calculations to avoid floating point issues.
3. **Never trust client-side fee calculations** — always recalculate `application_fee_amount` server-side before creating Checkout Session.
4. **Idempotency** — use `idempotency_key` on all Stripe create calls (PaymentIntent, Refund, Payout).
5. **Webhook handlers must be idempotent** — same event delivered twice should not create duplicate records. Check `event.id` against processed events table.
6. **Never expose Stripe secret key** — only use in server-side code (API routes, webhooks).
7. **Always verify webhook signatures** — `stripe.webhooks.constructEvent()` with `STRIPE_WEBHOOK_SECRET`.
8. **Use metadata extensively** — always attach `booking_id`, `guide_id`, `trip_id`, `transaction_type` to every Stripe object.
9. **Log everything** — every Stripe API call, webhook event, and state transition gets logged for debugging and audit.
10. **Feature flag new payment methods** — Vipps, Klarna, SEPA, Bancontact, iDEAL behind env var flags.
11. **Test with Stripe Test Clocks** — use test clocks for scheduled payments (auto-charge balance) to simulate time passing.
12. **Handle Stripe errors gracefully** — wrap all Stripe calls in try/catch, map to user-friendly error messages.
13. **Guide payout = trip_price - commission** — guide NEVER pays Stripe processing fees. Platform absorbs Stripe fees from its commission + service fee revenue.
14. **Service fee cap** — always enforce `Math.min(tripTotal * 0.05, 5000)` for service fee calculation. Never exceed €50 service fee.
15. **Payment tier awareness** — always check `guide_payment_settings.payment_tier` before creating checkout sessions. Direct Payment tier uses simple Stripe Checkout (no Connect, no transfer_data). Stripe Connect tier uses destination charges.
16. **Direct Payment bookings** — platform only controls booking fee refunds. Guide's portion is outside platform's control. Never promise angler a refund of guide's portion.
