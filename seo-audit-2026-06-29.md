## CRITICAL Issues

### C-1 — Legal Pages Return 404 (Immediate Trust & GDPR Risk)
**First principle (THINK):** Google penalises sites lacking terms and privacy — especially after GDPR. Consent Mode v2 is configured but there is no privacy policy page to link to.
**Finding:** `/terms` → 404, `/privacy` → 404.
**Impact:** E-E-A-T collapse (Trustworthiness), potential GDPR liability, Consent Mode v2 renders meaningless without a privacy page, Google Quality Raters flag the absence.
**Fix:** Create `/src/app/(public)/terms/page.tsx` and `/src/app/(public)/privacy/page.tsx`. Add to footer nav. Add to robots.txt Disallow exceptions list if currently blocked.
**Falsifiability check (ACCEPT):** Pages render 200, linked in footer, Privacy Policy linked from Consent Mode banner.
**Leading indicator (GROW):** Google Search Console → Coverage → Valid (monitor for `/terms` + `/privacy` indexed within 7 days of publishing).

---

### C-2 — IndexNow Not Implemented
**THINK:** Without IndexNow, Google and Bing learn about new content only on their next crawl cycle (days to weeks). Competitors who publish similar content get indexed first.
**Finding:** No IndexNow key file at `/[key].txt`, no IndexNow API calls in codebase.
**Impact:** Delayed indexing of new guide profiles and experience pages = lost early-ranking opportunity.
**Fix:** Generate IndexNow key (free at indexnow.org), add `[key].txt` to `/public/`, ping IndexNow endpoint from Supabase webhook or Next.js `revalidatePath` hook on content publish.
**ACCEPT:** New experience pages appear in Bing Webmaster Tools "URL Submission" within 24h of publish.
**GROW:** Track "Coverage → Newly Discovered" in GSC week-over-week.

---

### C-3 — Deprecated HowTo Schema on Homepage
**THINK:** Google deprecated HowTo rich results in September 2023. Serving deprecated schema wastes render budget and may confuse structured data validators.
**Finding:** `src/app/layout.tsx` or homepage contains HowTo JSON-LD schema.
**Impact:** No rich result rendered (schema is ignored by Google), wasted markup, potential future penalty if guidelines tighten.
**Fix:** Remove HowTo schema. Replace with ItemList (experience listing), or retain the content blocks as plain HTML.
**ACCEPT:** Rich Results Test at search.google.com/test/rich-results shows no HowTo errors.
**GROW:** Schema Validation errors in GSC → Enhancements.

---

### C-4 — Homepage Uses `force-dynamic` Instead of ISR
**THINK:** `export const dynamic = 'force-dynamic'` on the homepage means every request hits the server cold. Homepage is the highest-traffic, most link-equity-holding page — it should be statically generated or ISR-cached.
**Finding:** `src/app/(public)/page.tsx` (or root layout) exports `dynamic = 'force-dynamic'`.
**Impact:** TTFB inflated (estimated 800ms–2s), LCP degrades, Vercel compute costs spike, CWV scores suffer.
**Fix:** Remove `force-dynamic`. Use `export const revalidate = 60` (ISR) for the homepage. Extract dynamic personalization (if any) into Client Components with `Suspense`.
**ACCEPT:** Vercel Analytics shows homepage TTFB < 200ms after change.
**GROW:** CrUX LCP field data (28-day rolling) — target LCP ≤ 2.5s.

---

### C-5 — /trips Listing Page Has No H1 Tag
**THINK:** H1 is the single most important on-page ranking signal. A listing page without H1 tells Google there is no primary topic.
**Finding:** `/trips` page renders content but no `<h1>` element in DOM.
**Impact:** Page cannot rank for core category keywords ("guided fishing trips Norway").
**Fix:** Add `<h1>Guided Fishing Trips in Norway, Sweden, Iceland & Finland</h1>` as the first heading on the trips listing page.
**ACCEPT:** `curl https://fjordanglers.com/trips | grep -i "<h1"` returns one match.
**GROW:** GSC → Queries for "guided fishing trips norway" — track impressions rising within 4–6 weeks.

## HIGH Priority Issues

### H-1 — Guide Profile URLs Use UUID Instead of Slug
**Finding:** Guide pages are `/guides/a1b2c3d4-uuid` not `/guides/john-smith-norway`.
**Impact:** Zero keyword value in URL, difficult to share/remember, Google can't infer topic from URL structure.
**Fix:** Add `slug` column to `guides` table (unique, NOT NULL), generate slugs from `full_name` + country (e.g., `erik-larsen-norway`). Update all internal links to use slug. Add 301 redirect from UUID URL to slug URL.
**ACCEPT:** `curl -I https://fjordanglers.com/guides/erik-larsen-norway` returns 200. UUID URL returns 301.

---

### H-2 — Blog Content Is Static Array, Not DB-Driven
**Finding:** `src/app/(public)/blog/page.tsx` imports `BLOG_POSTS` from `@/lib/blog-data` (hardcoded array, not Supabase).
**Impact:** Blog posts cannot be added/edited without a code deploy. No dynamic sitemap coverage for new posts. No author pages possible. Limits E-E-A-T signals (no author schema, no publication date updates).
**Fix:** Migrate blog to Supabase `blog_posts` table (title, slug, content, author_id, published_at, hero_image_url). Update sitemap generator to pull dynamically. Add `BlogPosting` schema per post.
**ACCEPT:** New blog post published via Supabase dashboard appears in sitemap within 60s (ISR revalidate).

---

### H-3 — No Hreflang Tags Despite Multilingual Signals
**Finding:** `layout.tsx` declares `openGraph.alternateLocale: ['pl_PL', 'de_DE']` but no `<link rel="alternate" hreflang="...">` tags exist in `<head>`.
**Impact:** Google cannot serve Polish or German language variants to appropriate audiences. Polish anglers (primary customer base) may see English content in Google PL.
**Fix:** Either implement hreflang for pl/de/en with proper translated pages, or remove `alternateLocale` from OG to avoid false signal. Short-term: remove `alternateLocale`. Long-term: build `/pl/` and `/de/` routes with hreflang.
**ACCEPT:** `curl https://fjordanglers.com | grep hreflang` returns matching x-default + en + pl + de tags (or zero if cleaned up).

---

### H-4 — Guides Listing Page Title Missing Brand
**Finding:** `src/app/(public)/guides/page.tsx` title is `"Fishing Guides in Norway, Sweden, Iceland & Finland"` (52 chars) — no brand name.
**Impact:** Brand recall lost in SERP, CTR lower than branded snippets.
**Fix:** Use template: `"Fishing Guides in Norway, Sweden & Iceland | FjordAnglers"` (≤60 chars). Already configured in root metadata `template: '%s | FjordAnglers'` — ensure sub-page uses plain title string that feeds the template.
**ACCEPT:** SERP snippet for `site:fjordanglers.com/guides` shows `| FjordAnglers` in title.

---

### H-5 — FAQPage Schema: Rich Results Ended May 7, 2026
**Finding:** `src/app/experiences/[slug]/page.tsx` conditionally renders FAQPage JSON-LD.
**Impact:** Google officially ended FAQ rich results for all sites on May 7, 2026. No SERP feature is rendered. The schema still has value for AI/LLM citations — retain it but stop expecting SERP enhancement.
**Fix:** Do NOT remove FAQPage schema (AI citation benefit remains). Do NOT add new FAQPage for Google SERP ranking. Update internal docs/expectations. If using `QAPage` type for genuine user Q&A, that is appropriate.
**ACCEPT:** Documentation/comments in code clarify: "FAQPage retained for AI citation benefit only — not for Google rich results (ended May 7 2026)."

---

### H-6 — Experience Pages: Guide URL Uses UUID Not Slug
**Finding:** In TouristTrip schema: `url: \`https://fjordanglers.com/guides/${guide.id}\`` — UUID instead of slug.
**Impact:** Structured data carries UUID URLs which degrade trust signals in schema validators.
**Fix:** Once H-1 (guide slugs) is implemented, update `experiences/[slug]/page.tsx` TouristTrip schema to use `guide.slug`.
**ACCEPT:** Rich Results Test shows guide URL in TouristTrip schema as `/guides/erik-larsen-norway`.
**Dependency:** Blocked by H-1.

---

### H-7 — Consent Mode v2: All Denied by Default with No Consent Banner
**Finding:** `layout.tsx` sets all Consent Mode v2 signals to `denied` by default. No cookie consent banner is visible in code/audit.
**Impact:** If no banner appears, users never grant consent → Google Ads conversion modelling degraded → GTM fires in unconsented state (GDPR risk). If banner does exist (client-side rendered), it may not surface in SSR for crawlers.
**Fix:** Implement a server-side-rendered consent banner (or use a CMP like Cookiebot/CookieYes). Link to `/privacy` page (requires C-1 fix first). Ensure consent state persists in cookie.
**ACCEPT:** `curl https://fjordanglers.com | grep -i "cookie\|consent"` returns consent banner markup in HTML.
**Dependency:** Blocked by C-1 (privacy page must exist first).

## MEDIUM Priority Issues

### M-1 — No `llms.txt` File (GEO Gap)
**Finding:** `GET https://fjordanglers.com/llms.txt` → 404.
**Impact:** AI crawlers (GPT, Perplexity, Claude) have no structured permission/guidance for indexing FjordAnglers content.
**Fix:** Create `/public/llms.txt` with brand description, permitted content areas, and primary URL list. Reference: llmstxt.org standard.
**ACCEPT:** `curl https://fjordanglers.com/llms.txt` returns 200 with valid content.

---

### M-2 — Species Pages Have Thin Unique Content (×19 pages)
**Finding:** 19 species pages in sitemap. If these share a template with minimal variation, they risk being flagged as thin/duplicate content.
**Impact:** Potential quality penalty, dilutes crawl budget.
**Fix:** Each species page needs: unique intro paragraph (≥200 words about that species in Nordic context), specific regions where it's found, best season, recommended gear, 1 internal link to a relevant experience page, 1 internal link to a relevant guide. Minimum 400 words unique content per page.
**ACCEPT:** Content quality audit using `/seo content` on 5 random species pages — all pass thin-content threshold.

---

### M-3 — No Internal Linking Strategy Between Experiences and Guides
**Finding:** Experience pages and guide profile pages exist as isolated silos with minimal cross-linking.
**Impact:** PageRank doesn't flow between related content, topical authority stays fragmented.
**Fix:** On each experience page, add "Meet Your Guide" section linking to guide's profile. On each guide profile, list their available experiences. On species pages, link to relevant experiences featuring that species.

---

### M-4 — No `BlogPosting` Schema on Blog Posts
**Finding:** Blog posts exist but have no Article/BlogPosting JSON-LD schema.
**Impact:** No rich snippet eligibility (headline, author, datePublished), reduced AI citation value.
**Fix:** Add `BlogPosting` schema with: `headline`, `author` (Person), `datePublished`, `image`, `url`, `publisher` (Organization), `wordCount`. Link `author.url` to author bio page.
**ACCEPT:** Rich Results Test on any blog post shows valid Article schema.

---

### M-5 — Homepage Hero: Video Background Likely Hurts LCP
**Finding:** Homepage appears to use a video background in the hero section.
**Impact:** Video = large resource, delays LCP (Largest Contentful Paint). Estimated LCP: 3–5s on mobile.
**Fix:** Replace video hero with static image (WebP, preloaded with `<link rel="preload">`) on mobile. Use `loading="eager"` on LCP image. Consider `fetchpriority="high"`. Video can load progressively after LCP fires.
**ACCEPT:** PageSpeed Insights mobile LCP ≤ 2.5s after change.

---

### M-6 — Google Ads Script Loaded Without Async Optimization
**Finding:** Google Ads (AW-18008446689) script in layout. Using Next.js `Script` with `strategy="afterInteractive"` is correct, but GA/GTM scripts still block INP budget.
**Impact:** INP (Interaction to Next Paint) degraded by third-party script execution.
**Fix:** Audit all third-party scripts for INP impact. Use `scheduler.postTask` or `requestIdleCallback` for non-critical scripts. Consider Partytown for complete main-thread offloading.
**ACCEPT:** INP ≤ 200ms (Good) in CrUX field data.

---

### M-7 — Sitemap: 15 Guide Profile URLs Are UUID-Based
**Finding:** `sitemap.xml` lists `/guides/a1b2c3d4-uuid` format URLs.
**Impact:** After H-1 (guide slugs) is implemented, sitemap will need regeneration to reflect new slug URLs. Old UUID sitemap entries waste crawl budget.
**Fix:** After H-1 is complete, regenerate sitemap with slug-based guide URLs. Remove UUID-based entries.
**Dependency:** Blocked by H-1.

---

### M-8 — Missing `preconnect` for Supabase Domain
**Finding:** No `<link rel="preconnect">` for Supabase storage/API domain in layout.
**Impact:** First request to Supabase (images, data) pays full DNS + TLS handshake cost, adding 100–300ms to initial load.
**Fix:** Add to `layout.tsx`:
```html
<link rel="preconnect" href="https://xsilxmaiyyjgpxsalvet.supabase.co" />
<link rel="dns-prefetch" href="https://xsilxmaiyyjgpxsalvet.supabase.co" />
```

## LOW Priority Issues

### L-1 — Sitemap Priority Values Not Reflecting Actual Content Value
**Finding:** Species pages (`0.7`) and blog posts (`0.6–0.7`) have similar or higher priority than guide profiles (`0.8`). Guide profiles generate revenue; they should be highest non-homepage priority.
**Fix:** Reprioritize: homepage `1.0` → experience pages `0.95` → guide profiles `0.90` → species `0.70` → blog `0.65` → legal `0.3`.

---

### L-2 — No Author Bio Pages for Blog
**Finding:** Blog has no author profile pages.
**Impact:** E-E-A-T (Expertise/Authoritativeness) signal missing. Who writes the fishing journal?
**Fix:** Create `/guides/[slug]/blog` or `/authors/[slug]` pages. Even a minimal "About the Author" card improves E-E-A-T for content.

---

### L-3 — No Open Graph Images for Experience Pages
**Finding:** Experience pages may not have unique OG images — they likely inherit root layout OG.
**Impact:** Social shares show generic image, reducing CTR from social referrals.
**Fix:** In `generateMetadata` for experience pages, set `openGraph.images` from the experience's hero image URL.

---

### L-4 — Robots.txt Blocks `/book/` (May Block Checkout Schema)
**Finding:** `Disallow: /book/` in robots.txt.
**Impact:** If any schema or canonical content lives under `/book/`, it won't be indexed. This is probably intentional (checkout pages shouldn't be indexed), but verify no SEO content is under this path.
**Fix:** Verify no indexable content under `/book/`. If clear, no action needed.

---

### L-5 — No Structured Data on Guides Listing Page
**Finding:** `/guides` page has no JSON-LD schema.
**Fix:** Add `ItemList` schema listing all guides as `ListItem → Person`. This helps AI crawlers understand the page is a collection of fishing guides.

## ADDENDUM — Additional Critical Findings

### A-1 — Consent Mode v2 + Missing Privacy Page = Active GDPR Violation
The combination of: (a) collecting any data via GTM/GA4/Google Ads AND (b) having no privacy page AND (c) consent defaulting to `denied` with no visible banner to grant consent means the site is in an ambiguous GDPR state. Polish DPA (UODO) and Norwegian Datatilsynet both actively enforce. **Priority: Highest.**

### A-2 — Missing `<title>` on `/trips` Page
Related to C-5 (missing H1) — verify the `/trips` page also has a proper `<title>` tag. If `generateMetadata` is not implemented for this route, it falls back to the root template (`undefined | FjordAnglers`), which renders as ` | FjordAnglers` — an empty title.
